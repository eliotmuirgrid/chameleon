# Case-preserving storage (paths)

Optional depth for [Letter case (for paths and names)](../letter_case.md).

---

## Concept

**Case-preserving storage** means: the system **remembers** which letters you typed (upper or lower) when it **stores** or **shows** a name, while **matching** may still ignore case. Pair this with [Case-insensitive comparison](case_insensitive_comparison.md): you can get **one** underlying file whose spelling in listings matches how it was first created.

## Prerequisites

Read [Letter case (for paths and names)](../letter_case.md), then [Case-insensitive comparison](case_insensitive_comparison.md), when both ideas appear together.

## Where you see it

Typical **NTFS** and default **APFS** / **HFS+** behavior described in [Filesystem case on major desktop OS families](../../git/git_core_ignorecase/filesystem_case_by_os.md).
