# Common filesystems (index)

Hub: [Common filesystems (cross-platform)](../common/filesystems.md). Each file below is **one identity**—link here from OS or Git pages instead of stacking every format on one page.

## Pages

- [NTFS](ntfs.md) — usual default on Windows desktops; typically [case-insensitive](../../letter_case/case/insensitive/comparison.md), [case-preserving](../../letter_case/case/preserving/storage.md). Vocabulary: [Letter case (for paths and names)](../../letter/case.md).
- [APFS](apfs.md) — modern Apple filesystem on Mac internal drives; default consumer volumes often [case-insensitive](../../letter_case/case/insensitive/comparison.md), [case-preserving](../../letter_case/case/preserving/storage.md). [Letter case (for paths and names)](../../letter/case.md).
- [HFS+](hfs/plus.md) — older Apple filesystem, still common on externals; same [letter case](../../letter/case.md) picture as typical APFS for consumers.
- [ext4](ext4.md) — common native Linux layout; [case-sensitive](../../letter_case/case/sensitive/comparison.md) paths. Vocabulary: [Letter case (for paths and names)](../../letter/case.md).
- [XFS](xfs.md) — [journaled filesystem](../journaled/filesystem.md) common on Linux; [letter case](../../letter/case.md) like ext4 for ordinary paths. General pattern: [Journaling](../../../system/design/journal.md).
- [Btrfs](btrfs.md) — advanced Linux filesystem; [letter case](../../letter/case.md) like ext4 for ordinary paths ([case-sensitive](../../letter_case/case/sensitive/comparison.md)).
