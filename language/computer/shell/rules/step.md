# Rule: Write Steps Before Script

Before writing any shell script, write a clear step list of what the script must do.

The step list is the script documentation.

## Requirements

- Write steps in order, from start to finish.
- Keep each step simple and testable.
- Use this step list to guide implementation.
- Update the step list if script behavior changes.

## Why This Is Required

- It reduces [cognitive load](../../../../cognition/load.md).
- It keeps implementation aligned with intent.
- It makes review and maintenance faster.
