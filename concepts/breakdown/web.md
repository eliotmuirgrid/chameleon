# Recursive concept decomposition (break up into a web)

## Concept

When a page grows too wide, split it into smaller pages and link them.
Repeat until each page has one main idea.

You are not deleting complexity.
You are placing complexity into structure so people can follow one thread at a time.

## Why

- Lower [cognitive load](../system/design/cognitive_load/README.md)
- Better fit for [Simple English](../communication/simple_english.md)
- Better [separation of concerns](../system/design/separation_of_concerns/concept.md)

## Practice

Use the checklist: [Recursive decomposition — practice checklist](recursive_decomposition_practice.md).
Keep tool logic and environment rules separate.

## Takeaway

If a page needs long detours to stay correct, create or link a separate concept page instead of stretching the current page.
