# Singular path segments

## Concept

Use a **singular** noun (or short **identifier** without **spaces**, **hyphens**, or **underscores** in the segment) when the page or folder is **one topic**, **one mechanism**, or **one identity**—for example `ext4.md`, `entropy.md`, `equilibrium.md`. Singular names read like **labels** on a box: one job per path. They pair with the [two-layer detail pattern](../../../detail.md) (`topic.md` plus `topic/`) without sounding like a **miscellaneous pile**.

When a name wants **several everyday words**, do **not** pack them into one segment with `_` or `-`; see [No underscores](../../no/underscores/README.md) and [No hyphens](../../no/hyphens/README.md). Add **parent folders** or a **detail** file instead. When **plural** grammar would blur whether the path is **one hub** or **many items**, see [No plural](../../no/plural/README.md).
