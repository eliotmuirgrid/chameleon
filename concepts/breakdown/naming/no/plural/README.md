# No plural folder or file names

## Concept

**Plural** words in a path segment (`cases/`, `formats.md`, `filesystems/`) are **not** used under `concepts/`. A plural **promises cardinality**: “many of a kind,” often without saying **which** kind owns the page. That is **naming ambiguity**: readers cannot tell whether the segment is **one hub** about a topic, a **directory of peers**, or a **miscellaneous pile**. Singular segments read as **one label per job**; see [Singular](../../yes/singular/README.md).

This is the same **stability** goal as [No spaces in `concepts/` names](../spaces/README.md), [No hyphens](../hyphens/README.md), and [No underscores](../underscores/README.md): path text should not **overload** English grammar to carry structure that belongs in **folders** or **links**.
