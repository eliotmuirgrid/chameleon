# Concept - Inline Links

## Concept

When a related concept is already named in a sentence, use an inline link on that phrase instead of adding a separate link line later. That matches the [concept standard](../concept.md), fits the [two-layer detail pattern](detail.md) (links carry you to depth instead of long digressions in place), and pairs with [Simple English (80/20 language rule)](../communication/simple_english.md) so the linked phrase stays readable. The point is to keep [cognitive load](../system/design/cognitive_load/README.md) low: meaning and navigation land in one pass.

## Why

- Keeps the reader in flow by linking at the point of meaning.
- Reduces extra scanning between body text and link lists.
- Makes relationships between concepts clearer in context.

## Quick test

Ask:

1. Did I mention a concept name without linking it, even though a page exists?
2. Would moving a link inline make the sentence clearer?
3. Did I avoid duplicating the same link both inline and in a nearby list without a reason?
