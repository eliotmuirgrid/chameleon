# Path and file naming (concepts tree)

## Concept

Path names are part of the product.
Use names that are easy to read, easy to type, and stable over time.

## Rules

1. Do not use spaces, hyphens, underscores, or plural segment names in `concepts/` paths.
2. Prefer lowercase names.
3. Prefer one clear token per path segment.
4. If you need extra words, add folder depth instead of packing one segment.

See the rule pages in [Naming `no/`](naming/no.md) and the positive pattern in [Singular](naming/yes/singular/README.md).

## Why

- Less naming ambiguity
- Fewer broken assumptions in tools and scripts
- Clearer page boundaries

Related shell behavior: [Spaces in file names and the command line](../computer/filename_spaces_command_line.md).
