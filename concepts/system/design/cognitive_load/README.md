# Cognitive load

**Cognitive load** is how much thinking effort a task needs right now.

High cognitive load means you must hold many things in your head at once.  
Low cognitive load means the next step is obvious.

## System view

A system has lower cognitive load when it:

- has clear names,
- has clear steps,
- shows the current state,
- and reduces hidden choices.

## Core variable

- **Working memory demand** - how many active items a person must track at one time.

## Quick test

If you stop for 10 minutes, can you restart without re-learning everything?

- **Yes** -> load is probably low.
- **No** -> load is probably high.

## Why it matters

Lower cognitive load usually gives:

- fewer mistakes,
- faster learning,
- steadier execution.
