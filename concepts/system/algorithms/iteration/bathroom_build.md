# Iteration example - Building a bathroom

Building a bathroom with an iterative approach means making one change at a time, checking results, and learning before the next change.

In practice, a [slatboard](../../../home/slatboard.md) can serve as the structural base so mirrors, lighting, and accessories can be repositioned without tearing walls apart.

Instead of trying to finalize everything in one pass, you run short loops:

1. choose one small change,
2. implement it,
3. observe how it performs (function, cost, effort, quality),
4. adjust the next step.

This lowers risk because mistakes stay small and easier to correct.

It also improves decision quality because each step gives real feedback for the next one.

This turns the bathroom from a static install into an evolving system.

Even items that are not in their "final" position can still provide value: mounted components can be used, tested, and improved immediately.

Over time, repeated small corrections move the project toward a practical [equilibrium](../../design/equilibrium.md): stable function, acceptable cost, and manageable effort.
