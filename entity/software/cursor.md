# Cursor (editor)

**Summary:** Cursor is a code editor built around the same core experience as Visual Studio Code, with built-in artificial intelligence help for reading, editing, and navigating code. You can use it like a normal editor, then add assistant-driven workflows when useful.

---

## What it is

- **Editor first** — files, folders, search, terminal, and extensions.
- **Assistant inside the editor** — helps explain code, propose changes, and apply edits.
- **Project context** — can work with your repo structure and linked concept pages while you edit.

---

## Why teams use it

- Faster drafting and refactoring for routine changes.
- Easier onboarding: questions can be asked in-place while reading code.
- Better flow when documentation, data tables, and code live together.

That flow sits next to [GitHub, Bitbucket, and Git](../../computer/git/github/bitbucket/and/git.md) (local Git vs hosting) and to readable [Markdown table format](../../system/design/symmetry/markdown/table.md) in this repo.
